# AI Agent ESPer-Chan

Copyright git:mochimochi-man / H.N.うっ / X:@calorie0

## 使い方ガイド

小さなコンピューターに宿る、あなた専用のAIアシスタントです。

---

## 1. プロジェクト概要

ESPer-Chan(エスパーちゃん)は、Xiao ESP32S3 Sense※を使った「ローカルAIアシスタント」です。

ESP32S3をフロントエンドとし、自宅のPCで動作するLM Studio(大規模言語モデル)、whisper.cpp(音声->文字変換)、VOICEVOX(文字->音声変換)と連携させることで、AIをクラウドサービスに依存せず、音声で話しかけ、音声で応答する対話を楽しめます。

M5Stack Avatarを利用したカスタムフェイスを搭載し、話している間は口が動くリップシンクにも対応。「ねぇねぇ」と声をかけるだけで会話が始まります。

> ※現状、日本語ローカル専用版となっています。(グローバル版対応予定)
> ※一部の機能は汎用ESP32-S3でも動作することを確認しています。詳細は[9. ESP32-S3 Devkitでの動作](#9-esp32-s3-devkitでの動作)をご覧ください。

## 2. できること

| 機能 | 説明 |
|---|---|
| 音声会話 | 「ねぇねぇ」と話しかけると、音声を認識してAIが応答します |
| テキスト入力 | USBシリアルから文字を入力して、AIとチャットできます |
| 音声応答 | AIの返答はVOICEVOXで自然な日本語音声に変換されて読み上げられます |
| アバター表示 | 128x64の小型OLEDに顔が表示され、表情やリップシンクで反応します |
| WEBアバター表示 | 同一ネットワーク内PCのブラウザでアバターを表示できます。音声入力開始ボタンで音声入力も遠隔で操作できます。マイクは本体側を使用します |
| WEB設定画面搭載 | 自身をAPモードに切り替えることで、PCのブラウザから一部の設定を編集できます |
| 天気情報取得 | 天気予報API(livedoor天気互換)を利用して、今日、明日、明後日の天気情報を取得できます |
| 現在時刻取得 | NICTのAPIを利用して、現在時刻を取得できます |
| ローカルAI連携 | LM Studioで好きなLLM(大規模言語モデル)を選んで動かせます |
| ミュージックプレーヤー(Xiao ESP32S3 Sense用) | ミュージックプレーヤーモードに切り替えると、microSDカードに入れた音楽ファイル(MP3/MP4/WAV)を再生できます |
| フェイスディテクション(顔認識)搭載WEBカメラ(Xiao ESP32S3 Sense用) | カメラモードに切り替えると、フェイスディテクション(顔認識)機能付きWEBカメラとして利用できます。認識情報は指定した間隔でJSONで出力します |
| 各種コマンド | スピーカーテスト、マイクテスト、TTSテストなど、機能確認用コマンドが使えます |

## 4. セットアップ手順

### ■ Step 1: Arduino IDEの準備

1. Arduino IDE(2.x推奨)をインストールします。
   https://www.arduino.cc/en/software

2. Arduino IDEを開き、「ファイル」→「環境設定」をクリックします。

3. 「追加のボードマネージャーのURL」に以下のURLを貼り付けます:
   `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`

4. 「OK」をクリックして保存します。

5. 「ツール」→「ボード」→「ボードマネージャー」を開き、「ESP32 by Espressif Systems」を検索してインストールします。

6. 「ツール」→「ボード」→「XIAO_ESP32S3」(または「ESP32S3 Dev Module」)を選択します。

### ■ Step 2: 必要なライブラリをインストール

Arduino IDEの「スケッチ」→「ライブラリをインクルード」→「ライブラリを管理」から、以下のライブラリを検索・インストールしてください。

- **M5Unified**(M5Stack公式)
- **M5UnitGLASS2**(M5Stack公式)
- **M5Stack-Avatar**(meganetaaan製)
- **ArduinoJson**(Benoit Blanchon製)

> ※M5Stack-Avatarインストール後、Face.hの手動修正が必要になります。`Arduino/libraries/M5Stack_Avatar/src/Face.h` を2か所、以下のように修正してください。
>
> 1. `void draw(DrawContext *ctx);` → `virtual void draw(DrawContext *ctx);`
> 2. `private:` → `protected:`

### ■ Step 3: PC側サーバーの起動(Windows GUI版)

#### --- LM Studio(LLMサーバー) ---

1. ブラウザで https://lmstudio.ai/ を開き、Windows版をダウンロードします。

2. ダウンロードしたインストーラーを実行し、画面の指示に従ってインストールします。特に難しい設定はありません。

3. LM Studioを起動します。初回起動時にモデルをダウンロードする画面が表示されます。

4. 使いたいモデルを検索してダウンロードします。
   - 例: `lfm2.5-1.2b-instruct-GGUF` ※config.hのデフォルト設定
   - 例: `phi-4-mini-instruct`
   - 例: `qwen3-1.7b`
   - 例: `gemma-4-e2b`

5. 左下ギヤアイコンからSettingsを開き、Developerから「ローカルLLMサービスを有効にする」を有効にします。一度終了します。

6. 再度起動し、タスクトレイのLM Studioアイコンから「Start Server On Port 1234...」をクリックします。

7. サーバーが起動したら、そのままLM Studioは最小化しておいてください。

#### --- whisper.cpp(STTサーバー) ---

1. ブラウザで https://github.com/ggml-org/whisper.cpp/releases を開きます。

2. 画面下部、最新リリースの「Assets」から「whisper-bin-x64.zip」など、Windows用のZIPファイルをダウンロードします。

3. ダウンロードしたZIPを展開し、フォルダを任意の場所に配置します。
   - 例: `C:\whisper.cpp-bin`

4. モデルファイル(ggml-small.bin 推奨)を3.の下位のフォルダに入れます。
   - 例: `C:\whisper.cpp-bin\models`
   - モデルは https://huggingface.co/ggerganov/whisper.cpp/tree/main からダウンロードできます。

5. PowerShellを開き、3.のフォルダに移動します:
   ```
   cd C:\whisper.cpp-bin
   ```

6. サーバーモードで起動します:
   ```
   ./whisper-server -m ./models\ggml-small.bin --host 0.0.0.0 --port 8080 -l ja
   ```
   「HTTP server started」のメッセージが出れば成功です。このウィンドウは閉じないでください。

#### --- VOICEVOX(TTSサーバー) ---

1. ブラウザで https://voicevox.hiroshiba.jp/ を開きます。

2. 「ダウンロード」ボタンをクリックし、Windows版(インストーラー版)をダウンロードします。

3. ダウンロードしたインストーラーを実行し、画面の指示に従ってインストールします。特に難しい設定はありません。

4. インストールが完了したら、VOICEVOXを起動します。

5. 初回起動時に「音声ライブラリのダウンロード」が始まります。四国めたん(話者ID 2)がデフォルトで使用されるため、この話者のデータがダウンロードされていればOKです。一度終了します。

6. PowerShellを開き、VOICEVOXのインストールフォルダに移動します:
   ```
   cd C:\Users\(ユーザー名)\AppData\Local\programs\VOICEVOX\vv-engine
   ```
   `\AppData`は不可視属性なのでエクスプローラーの表示タブ→表示から隠しファイルをONにしてください。

7. VOICEVOXをサーバーモードで起動します。このウィンドウは閉じないでください。
   ```
   ./run.exe --host xxx.xxx.xxx.xxx
   ```

> ※ ESP32とPCは「同じWi-Fiネットワーク」及び「パブリックネットワーク」に接続している必要があります。また、Windowsファイアウォールでポート1234、8081、50021がブロックされていないか確認してください。

### ■ Step 4: config.h の設定

プロジェクトフォルダ内の「config.h」をテキストエディタで開き、自宅の環境に合わせて編集します。
WiFi設定など一部の設定項目は、APモードのTwenty-Chanに接続したPCのブラウザから設定することも可能です。詳しくは別項「[7. APモードによるWEB設定](#7-apモードによるweb設定)」をご覧ください。

**--- WiFi設定 ---**
```cpp
#define WIFI_SSID       "Your_WiFi_SSID"
#define WIFI_PASSWORD   "Your_WiFi_Password"
```

**--- LM Studio ---**
```cpp
#define LMSTUDIO_HOST   "xxx.xxx.xxx.xxx"
#define LMSTUDIO_PORT   1234
```

**--- whisper.cpp ---**
```cpp
#define WHISPER_HOST    "xxx.xxx.xxx.xxx"
#define WHISPER_PORT    8080
```

**--- VOICEVOX ---**
```cpp
#define TTS_HOST        "xxx.xxx.xxx.xxx"
#define TTS_PORT        50021
```

**--- モデル選択 ---**

使用するモデルの行の先頭の「//」を削除し、他の行は「//」を付けてコメントアウトしてください。

```cpp
// #define MODEL_NAME      "phi-4-mini-instruct"
// #define MODEL_NAME      "qwen3-1.7b"
#define MODEL_NAME      "lfm2.5-1.2b-instruct"
```

> HOST名(PCのIPアドレス)の確認方法:
> コマンドプロンプトで「ipconfig」と入力し、「IPv4 アドレス」の値を確認してください。

### ■ Step 5: コンパイルと書き込み

1. Arduino IDEで.inoを開きます。
2. 「ツール」→「ボード」で「XIAO_ESP32S3」を選択します。
3. 「ツール」→「ポート」でESP32S3が接続されているCOMポートを選択します。
4. 「ツール」メニュー下部のボードの設定でPSRAMを「OPI PSRAM」に設定します。
5. 左上の「→」(書き込み)ボタンをクリックします。
6. コンパイルが完了し、「書き込み完了」と表示されたら成功です。
7. シリアルモニタ(右下の虫眼鏡アイコン)を開き、ボーレートを「115200」に設定します。
8. 一度ESP32S3を再起動し、最後に「👤 あなた: 」と表示されればOKです。

## 5. 使い方

### ■ 起動

正常に起動すると次のような待機状態になります。

```
========================================
  AI Agent ESPer-Chan
  ESP32S3 x LM Studio + whisper.cpp + VOICEVOX
========================================
👤 あなた:
```

### ■ テキスト入力モード

シリアルモニタから文字を入力してEnterを押すと、AIに送信されます。
AIの応答はテキストで表示され、同時に音声でも読み上げられます。

```
👤 あなた: こんにちは
========================================
🤖 ESPer-Chan: こんにちは!元気?何か話したいことある?
========================================
```

### ■ 音声会話モード(ウェイクワード)

「ねぇねぇ」と声をかけると、数秒後にウェイクワードが検出されて音声入力モードに入ります。驚いたような顔になり、LEDが少し点滅し、ピピッと音が鳴ったら合図です。

そのまま話しかけると、音声が認識されてAIが応答します。

話しかけている間は無表情、サーバーとの処理中は笑った表情になります。

応答内容が長すぎると、途中で音声が切れることがあります。

```
✨ ウェイクワードを検出しました
📝 今日の気分はどう?
========================================
🤖 ESPer-Chan: 今日はいい気分だよ!お出かけしようか
========================================
```

音声入力で天気情報を聞くこともできます。その場合は、次のようにお話しください。

> "今日の天気" または "明日の天気" または "明後日の天気"

> ※天気情報の取得には、個人の善意で運用されているAPIを使用しています。常識を逸脱して天気情報の取得を繰り返す行為はおやめください。

また、音声入力で現在時刻を取得することもできます。その場合は、次のようにお話しください。

> "現在の時刻" または "今、何時"

### ■ ミュージックプレーヤーモード(Xiao ESP32S3 Sense用)

ミュージックプレーヤーモード(以下、MPモード)に切り替えると、microSDカードに入れた音楽ファイル(MP3/MP4/WAV)を再生できます。

MPモードに切り替えるには、シリアルモニタから/musicコマンドを入力するか、音声入力で"ミュージックモード"または"ミュージックプレイヤー"とお話しください。

MPモードに切り替えると、最初に曲名リスト(INDEX)を作成します。プレーヤーは、この曲リストに基づいて曲を再生します。曲を追加した場合は、/indexコマンドで曲リストを再構成してください。

曲を再生するには、/play(曲リスト順)、/random(ランダム)、/artist(アーティスト)コマンドを使用してください。

そのほか、MPモードでは専用のコマンドが使用できます。詳細は/helpを参照してください。

なお、MPモードでは音声入力が動作しません。通常モードに戻るには、シリアルモニタから/exitコマンドを入力するか、WEB画面のボタンからお戻りください。

### ■ フェイスディテクション(顔認識)搭載WEBカメラ(Xiao ESP32S3 Sense用)

カメラモードに切り替えると、フェイスディテクション(顔認識)搭載のWEBカメラとして利用できます。

カメラモードに切り替えるには、シリアルモニタから/camコマンドを入力するか、音声入力で"カメラ"とお話しください。

カメラモードに切り替えると、自動的に再起動し。WEB画面がカメラモードに切り替わります。STARTを押すとカメラが起動します。

> ※特定の人物を認識する機能はありません。
> ※カメラモード中は、指定した秒数ごとにJSONで認識情報をシリアルに出力します。ただし、タクトスイッチを接続する場合はTX/RXポートが使用できませんので注意してください。
> ※カメラモードではディスプレイが動作しません。通常モードに戻るには、シリアルモニタから/exitコマンドを入力するか、WEB画面のボタンからお戻りください。
> ※カメラモードは、/exitで通常モードに戻らない限り、カメラモードを維持します。カメラモードのまま再起動してもカメラモードのままになりますので、ご注意ください。

### ■ コマンド一覧

通常モード時、シリアルモニタから以下のコマンドを入力できます。

| コマンド | 説明 |
|---|---|
| `/voice` | 音声会話(マイク→LLM→読み上げ) |
| `/say <text>` | テキストを読み上げ |
| `/music` | ミュージックプレーヤーモード起動 |
| `/camera` `/cam` | ウェブカメラモード起動 |
| `/today` | 今日の天気予報 |
| `/tomorrow` | 明日の天気予報 |
| `/dat` | 明後日の天気予報 |
| `/time` | 現在時刻を読み上げ(NICT) |
| `/status` | WiFi・メモリ等のステータス表示 |
| `/clear` | 会話履歴をクリア |
| `/reset` | 再起動 |
| `/ap` | APモード(WiFi設定) |
| `/initialize` | 設定をデフォルト値に戻す |
| `/beep` | ビープ音テスト |
| `/mic` | マイクテスト(3秒録音) |
| `/stt` | STTテスト(録音→文字変換) |
| `/tts` | TTSテスト(読み上げ確認) |

## 6. カスタマイズ

### ■ AIアシスタントの名前を変える

config.hのPROJECT_NAMEとSPEAK_NAMEを変更してください。

```cpp
#define PROJECT_NAME    "My-Chan"
#define SPEAK_NAME      "まいちゃん"
```

SPEAK_NAMEはAIが読み上げる際に使用する名前です。

### ■ 性格を変える

config.hのSYSTEM_PROMPTを編集してください。

```cpp
#define SYSTEM_PROMPT   "あなたは「まいちゃん」という名前の小さな可愛いAIアシスタントです。親しみやすくて少しお茶目な性格です。短くて分かりやすい日本語で返答してください。"
```

### ■ AIモデルを変える

config.hのモデル選択部分で、使いたいモデルのコメントを外します。

```cpp
// #define MODEL_NAME      "phi-4-mini-instruct"
// #define MODEL_NAME      "qwen3-1.7b"
#define MODEL_NAME      "lfm2.5-1.2b-instruct"
```

LM Studio側で対応するモデルを読み込んでおく必要があります。

## 7. APモードによるWEB設定

config.hの一部は、Webによる設定も可能です。WEBによる設定を行うには、ESPer-ChanをシリアルモニタでAPモードに変更し、APモードに接続したPCからブラウザ経由でアクセスする必要があります。

1. **APモードに切り替える**

   WiFi接続の設定がない場合や接続先に接続できない場合、シリアルモニタから/apコマンドを入力するとAPモードに切り替わります。

2. **PCをAPモードのESPer-Chanに接続する**

   APモードに切り替えると接続先が表示されますので、PCを表示された接続先に接続してください。DHCPサーバー機能もありますのでIPアドレスの設定は不要です。

3. **ブラウザからWeb設定画面にアクセスする**

   ブラウザから指定されたIPアドレスにアクセスしてください。設定終了後は、PCの接続先を元の接続先に戻してください。

## 8. タクトスイッチの接続

タクトスイッチを接続すると、通常モード→ミュージックモード→カメラモードの切り替えボタンとして使えます。

接続するときは、以下を参考にしてください。

1. config.hの`#define USE_USB_CDC 0` を `1`
2. GPIO43とGNDの間にタクトスイッチを接続
3. 書き込み時にUSB CDC On Boot: を Enabledにして書き込み

> ※タクトスイッチを接続するとTX/RXポートが使用できなくなるため、カメラモードのJSON出力をシリアル経由でモニタリングするような用途には利用できません。
> ※USB CDC On Boot:Enabledで書き込んだ後、再度書き込みするときは、本体のBOOTボタンを押しながら電源を再接続してください。

## 10. トラブルシューティング

**Wi-Fi接続失敗**
config.hのSSID/パスワードが正しいか確認してください。
PCとESPer-Chanが同じWi-Fiネットワークに接続しているか確認してください。

**音声認識しない**
whisper.cppサーバーが起動しているか確認してください。
Windowsファイアウォールでポート8080がブロックされていないか確認してください。

**音声が出ない**
VOICEVOXエンジンが起動しているか確認してください。
MAX98357Aのピンやスピーカーが正しく接続されているか確認してください。

**顔が表示されない**
ディスプレイのピンが正しく接続されているか確認してください。
Arduino IDEでM5Unified、M5UnitGLASS2、M5Stack-Avatarライブラリがインストールされているか確認してください。

**応答が遅い**
使用しているLLMのサイズが大きすぎないか確認してください。
軽量モデル(1B〜4B程度)を推奨します。
